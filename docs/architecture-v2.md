# WorkshopOS v2 Architecture

## Джерело правди

- `data/` — структурована база в JSON.
- `docs/` — людська документація в Markdown.
- Старі Markdown-файли з `inventory/` залишаються як історія/чернетка, але нове джерело правди — `data/`.

## Класи об'єктів

| Префікс | Тип |
|---|---|
| TOOL | Ручний / переносний електроінструмент |
| MACH | Верстати та стаціонарне обладнання, включаючи пилососи і компресори |
| ACC | Оснастка: диски, ножі, фрези, свердла, біти |
| BAT | Акумулятори |
| CHR | Зарядні пристрої |
| DC | Dust Collection: циклони, шланги, адаптери, труби, заслінки |
| SYS | Акумуляторні / інструментальні системи |
| STORE | Зберігання |
| MEAS | Вимірювальний інструмент |
| CONS | Витратники |
| PROJ | Проєкти |
| PUR | Покупки |
| MNT | Обслуговування |

## Підтверджена логіка

- Пилосос = `MACH`.
- DeWALT Cyclone = `DC`, не `TOOL` і не `MACH`.
- Зарядка = `CHR`, не `TOOL`.
- Акумулятор = `BAT`, не `TOOL`.

## Основні зв'язки

- `BAT ⇄ CHR`
- `TOOL ⇄ BAT`
- `TOOL ⇄ ACC`
- `MACH ⇄ ACC`
- `TOOL ⇄ DC`
- `MACH ⇄ DC`
- `PROJ ⇄ TOOL / MACH / ACC`
- `PUR ⇄ будь-який об'єкт`
- `MNT ⇄ TOOL / MACH`

## Правила

1. Один об'єкт = один JSON-файл.
2. Усі зв'язки тільки через ID.
3. Не дублювати інформацію про інші об'єкти.
4. Якщо даних немає — `null` або `unknown`.
5. Не додавати непідтверджені моделі.
